# ChinaTechOS（手机维修后台 MVP）需求文档大纲

## 0. 项目目标

- 用后台系统替代 Excel 台账
- 双轨工单：快速维修 / 留机维修
- WhatsApp 模板生成 + 留痕（wa.me）
- 多门店预埋（store_id 隔离）
- 可追溯：时间线（order_events）
- Dashboard 待办（48h 待确认 / 5 天未取件 / paused）

## 1. 账号与门店

- 登录（账号/密码 + 门店代码）
- 角色权限：frontdesk / technician / manager
- 门店参数：48h、5 天规则

## 2. 客户与设备

- 客户列表/搜索（电话/姓名）
- 客户新增/编辑/软删除
- 设备挂载到客户：新增/编辑/软删除
- 电话规范化 phone_e164（默认 +39）

## 3. 工单（核心）

- 工单列表：搜索（publicNo/电话/姓名）+ 状态/类型/技师/结清筛选
- 新建工单：手机号 + 品牌型号 + 问题必填
- 双轨状态流转 + 校验
- 报价确认：waiting_approval 同意/拒绝（拒绝 → cancelled）
- 付款摘要：订金/尾款/is_paid
- 交付/完结
- 软删除

## 4. 消息中心（模板 + 留痕）

- 模板库 CRUD（manager）
- 工单内生成消息：渲染变量 → wa.me → 写入 message_logs
- message_logs：opened / marked_sent
- 发送记录列表：按门店、日期、模板筛选

## 5. 时间线与审计追溯

- order_events：创建、状态变更、报价发送、确认回写、付款更新、交付、完结
- 详情页时间线组件

## 6. Dashboard 待办

- 待确认超时（48h）
- 超期未取件（5 天）
- paused / 今日新建 / 今日完结
- 卡片跳转到筛选后的工单列表

## 7. 设置

- 门店信息（地址/营业时间/map_url/store_code）
- 自动化参数（48h、5 天）
- 模板库维护入口

## 8. 部署与运行

- GitHub → Vercel
- Vercel Postgres（或 Vercel 集成 Neon）
- 环境变量：POSTGRES_URL / AUTH_SECRET / AUTH_TRUST_HOST
